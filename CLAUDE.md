# Matcher — Chrome Extension for CMS Highlighting

A Chrome extension that highlights and categorizes words/phrases in BazaarVoice CMS reviews using a configurable dictionary. Designed for moderators to quickly spot retailers, shipping issues, profanity, and other flagged terms.

## Project Structure

```
extension/          # Chrome extension source (Manifest V3)
  matcher-core.js   # Pure matching engine (browser build)
  content.js        # DOM traversal and span-based highlighting
  popup.js          # Quick-toggle popup UI
  options.js        # Full settings/category management page
  background.js     # Service worker (storage, context menus)
  highlight.css     # Highlight span styles
tools/
  matcher.js        # Matching engine (Node.js/CommonJS build)
  test_assertions.js  # Primary test suite (pass/fail)
  test_real.js      # Real dictionary integration tests
  convert.js        # Converts HighlightThis backup JSON → matcher format
cms-fake/
  index.html        # Mock CMS UI for local testing without production
```

## Running Tests

```bash
# Primary test suite — run this after any changes to the matching engine
node tools/test_assertions.js

# Real dictionary integration tests
node tools/test_real.js
```

Both test files print colored pass/fail output. No test framework or install needed.

## Key Architecture Notes

### Matching Engine (`matcher-core.js` / `tools/matcher.js`)
- Both files contain the same logic — `matcher-core.js` exports `window.MatcherEngine`, `matcher.js` exports a CommonJS module.
- **If you change one, change the other.**
- Compiles word lists into chunked RegExp arrays for performance.
- Pattern syntax:
  - `CS:word` — case-sensitive
  - `//word//` — exact whole-word only
  - `word*` — wildcard (trailing)
  - Default — boundary-aware, case-insensitive

### Overlap Resolution
Exact matches beat wildcards; higher-priority categories win ties. Overlapping spans are resolved before DOM insertion.

### Content Script (`content.js`)
- Walks text nodes, skips nodes shorter than `minPatternLen` characters.
- Route guard is cached per navigation to avoid redundant checks.
- MutationObserver deduplicates rapid DOM changes.
- Some CMS routes (e.g., moderation queue) are blocked from highlighting.

### Dictionary (`converted_dictionary.json`)
- 150KB, ~4000+ words across 7+ categories plus an ignore list (~39 phrases).
- Generated by `tools/convert.js` from a HighlightThis backup file.
- The ignore list suppresses common false positives (e.g., "easy to use", "store front").

## Loading the Extension Locally

1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked** → select the `extension/` folder
4. For CMS-free testing, open `cms-fake/index.html` in a browser tab

## Converting a Legacy Dictionary

```bash
node tools/convert.js <path/to/HighlightThis_backup.json>
# Outputs: converted_dictionary.json
```
